# Customer Clauses

- **Возврат оплаты за сертификаты:** При возврате подарочного сертификата необходимо, по возможности, возвращать средства на то же платёжное средство, что использовалось при покупке; если это невозможно, требуется запросить у клиента актуальные платёжные реквизиты.
- **Единый поток состояний:** Все состояния объектов `GiftCertificate` (включая выпуска, списание, возвраты, уведомления, блокировки) публикуются в общем топике `GiftCertificates.Stream`; потребители фильтруют сообщения по статусу и каналу.
- **Использование существующих потоков заказов и платежей:** Каналы (ИМ, POS) отражают операции с сертификатами через `All_SalesOrder` и `All_Payments`; прямые HTTP-запросы к Payments не допускаются.
- **Доступ к CVC:** CRM не хранит CVC у себя и обязана получать код через API сервиса процессинга при каждом запросе.
